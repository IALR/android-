\section{Project Overview}
\subsection{Title}
\textbf{RoboConnect: Centralized Robotic Fleet Management}

\subsection{Short Description}
RoboConnect is an Android application paired with a heterogeneous robot fleet. The app discovers and pairs robots over Bluetooth/Wi-Fi, stores credentials for fast reconnects, and offers direct command/control. Robots expose a pairing button to enter pairing mode and accept app commands. The system addresses multi-robot management with a centralized UI, role-based access, and persistent automated connectivity.

\subsection{Target Users}
\begin{itemize}[noitemsep]
  \item Standard users: individuals controlling one or more robots (home automation, workshops, education); access shared robots.
  \item Robot administrators: users who add robots; full control, deletion, and permission management.
\end{itemize}

\section{Non-Functional Requirements}
\begin{itemize}[noitemsep]
  \item Stable Bluetooth/Wi-Fi communication during commands and auto-reconnect.
  \item Role/permission enforcement: only authenticated, authorized users can command robots.
  \item Intuitive navigation and clear UI (robust lists, easy command access).
  \item Responsive performance: fast loading of main screens and status updates.
  \item Data persistence via local DB or remote API/Firebase for continuity.
\end{itemize}

\section{Screens and Navigation}
\begin{itemize}[noitemsep]
  \item \textbf{Login/Home}: Enter credentials, sign in, proceed to Robot List.
  \item \textbf{Robot List}: See owned/shared robots with status; tap robot → Control; tap “+ Pair/Add” → Pairing; link to Settings/Admin.
  \item \textbf{Pairing}: Shows robots in pairing mode; select/register a robot; returns to Robot List.
  \item \textbf{Control Page}: Sliders/buttons/joysticks send commands; shows status; disconnect returns to Robot List.
  \item \textbf{Settings/Admin}: Manage permissions; delete robot; navigate back to Robot List or Control.
\end{itemize}
